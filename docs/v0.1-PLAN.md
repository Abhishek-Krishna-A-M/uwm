## File Responsibilities

### `build.zig`

- Define `uwm` executable target
- Use `wayland` module for protocol XML code generation
- Dependencies: `wayland-client` (C library via Zig Translation), `xkbcommon` (for keysym constants)
- Generate client-side C headers from XML protocol files
- Link `wayland-client`, `xkbcommon`, `c`

### `build.zig.zon`

- Declare `wayland-client` as a system library dependency
- Or use a Zig package for wayland (e.g., `zig-wayland`)

### `src/main.zig`

Responsibility: Application entry point and event loop.

```
main()
  ├── wl_display_connect(null) → display
  ├── registry = Registry.init(display)
  ├── registry.roundtrip()     // discover globals
  ├── assert(registry.wm != null)
  ├── assert(registry.xkb_bindings != null)
  ├── wm = Wm.init(registry.wm.?, registry.xkb_bindings.?)
  ├── event loop:
  │     while (true)
  │         wl_display_dispatch(display)
  └── cleanup
```

### `src/Registry.zig`

Responsibility: Bind to `wl_registry`, discover `river_window_manager_v1` and `river_xkb_bindings_v1` globals.

State:
- `wm_global: ?wl.proxy.WlRegistry` — will be bound to `river_window_manager_v1`
- `xkb_global: ?wl.proxy.WlRegistry` — will be bound to `river_xkb_bindings_v1`

Methods:
- `init(display)` — `wl_display_get_registry()`, set listener
- `roundtrip()` — `wl_display_roundtrip()` to populate globals
- `global_handler(name, interface, version)` — match on `"river_window_manager_v1"` (v≥4), `"river_xkb_bindings_v1"` (v≥1)

Listener callback on `global` event:
- Check `interface` string against known protocols
- Call `wl_registry_bind()` to create proxy at appropriate version
- Store proxy in Registry struct fields

### `src/Wm.zig`

Responsibility: `river_window_manager_v1` event listener. Central dispatch for manage/render sequences.

State:
- `wm: *river_window_manager_v1` — the bound proxy
- `xkb_bindings: *river_xkb_bindings_v1` — keyboard binding protocol
- `outputs: std.TailQueue(Output)` — tracked outputs
- `windows: std.TailQueue(Window)` — tracked windows
- `seats: std.TailQueue(Seat)` — tracked seats

Listener callbacks:
- `.unavailable` — print error, exit(1)
- `.finished` — exit(0)
- `.window` — create Window, get_node, add listener, push to list
- `.output` — create Output, add listener, push to list
- `.seat` — create Seat, add listener, push to list
- `.manage_start`:
    1. Remove dead windows (closed = true)
    2. Remove removed outputs/seats
    3. For each new window: propose_dimensions(0,0), set_position(0,0)
    4. For each new seat: create xkb bindings (Super+Space → spawn), enable them
    5. For each seat: set focus (last window in list, or clear)
    6. `manage_finish()`
- `.render_start`:
    1. `render_finish()`
- `.session_locked` — ignore
- `.session_unlocked` — ignore

Methods:
- `init(wm_proxy, xkb_proxy)` — store proxies, add listener
- `manageStart()` — see above
- `renderStart()` — see above
- `spawnFoot()` — `fork()` + `execlp("foot", "foot", null)`

### `src/Window.zig`

Responsibility: Wrap `river_window_v1` proxy and `river_node_v1`.

State:
- `proxy: *river_window_v1`
- `node: *river_node_v1` — render list node (from `get_node`)
- `new: bool` — newly created, needs initial geometry
- `closed: bool` — received closed event, pending cleanup
- `x: i32, y: i32` — current position
- `width: i32, height: i32` — current dimensions

Listener callbacks:
- `.closed` → set `closed = true`
- `.dimensions` → update `width`, `height`
- `.dimensions_hint` → ignore
- `.app_id` → ignore (store in future)
- `.title` → ignore (store in future)
- `.parent` → ignore
- `.decoration_hint` → ignore
- `.pointer_move_requested` → ignore for now
- `.pointer_resize_requested` → ignore for now
- `.fullscreen_requested` → ignore
- `.unreliable_pid` → ignore
- `.identifier` → ignore
- All others → ignore

Methods:
- `init(proxy)` — store proxy, call `get_node()`, set `new = true`
- `setPosition(x, y)` — `river_node_v1.set_position()`
- `deinit()` — destroy proxy

### `src/Output.zig`

Responsibility: Wrap `river_output_v1` proxy.

State:
- `proxy: *river_output_v1`
- `removed: bool`
- `x: i32, y: i32` — output position
- `width: i32, height: i32` — output dimensions

Listener callbacks:
- `.removed` → set `removed = true`
- `.wl_output` → ignore
- `.position` → store x, y
- `.dimensions` → store width, height

Methods:
- `init(proxy)` — store proxy, set listener
- `deinit()` — destroy proxy

### `src/Seat.zig`

Responsibility: Wrap `river_seat_v1` proxy. Manage bindings and focus.

State:
- `proxy: *river_seat_v1`
- `new: bool` — newly created, needs bindings
- `removed: bool`
- `focused: ?*Window` — currently focused window
- `bindings: std.TailQueue(Binding)` — active bindings

Listener callbacks:
- `.removed` → set `removed = true`
- `.wl_seat` → ignore
- `.pointer_enter` → focus window (match by proxy)
- `.pointer_leave` → clear hover focus
- `.window_interaction` → raise focused window (via place_top)
- `.shell_surface_interaction` → ignore
- `.op_delta` → ignore (no interactive ops yet)
- `.op_release` → ignore
- `.pointer_position` → ignore

Methods:
- `init(proxy)` — store proxy, set listener, set `new = true`
- `createXkbBinding(xkb_bindings, keysym, modifiers)` — call `river_xkb_bindings_v1.get_xkb_binding()`, enable, store in binding list
- `focusWindow(window)` — `focus_window(proxy)`, `node.place_top()`
- `clearFocus()` — `clear_focus(proxy)`
- `deinit()` — destroy all bindings, destroy proxy

### `src/Binding.zig`

Responsibility: Wrap `river_xkb_binding_v1` or `river_pointer_binding_v1`.

State (tagged union):
- `.xkb: struct { proxy: *river_xkb_binding_v1, keysym: u32, modifiers: u32 }`
- `.pointer: struct { proxy: *river_pointer_binding_v1, button: u32, modifiers: u32 }`

Listener callbacks:
- `.pressed` → notify seat of pending action
- `.released` → ignore

Methods:
- `initXkb(proxy, keysym, modifiers)` — create xkb binding wrapper
- `initPointer(proxy, button, modifiers)` — create pointer binding wrapper
- `enable()` — `proxy.enable()`
- `deinit()` — `proxy.destroy()`

## Control Flow (v0.1)

```
1. User runs: uwm
2. UWM connects to Wayland display
3. UWM discovers river_window_manager_v1 + river_xkb_bindings_v1
4. UWM adds river_window_manager_v1 listener
5. Event loop starts (wl_display_dispatch blocks)

6. River sends: [output events, seat event, window events]
   → UWM creates Output, Seat, Window objects
   → Each set to "new = true"

7. River sends: manage_start
   → Clean up any closed/removed objects (none yet)
   → For new Window: propose_dimensions(0,0), set_position(0,0)
   → For new Seat: create xkb binding (Mod4+Space → spawn foot), enable
   → No windows yet → seat.clearFocus()
   → manage_finish()

8. River sends: (nothing to render, no windows) → render_start → render_finish

9. User presses Super+Space
   → River matches against registered binding
   → River sends pressed event on river_xkb_binding_v1
   → UWM stores "pending_action = spawn_foot" on seat
   → River sends manage_start
   → In seat management: dispatch spawn_foot
       → fork() + execlp("foot", "foot", null)
   → manage_finish()

10. River sends: window event for foot
    → UWM creates Window, sets new = true
    → manage_start fires
    → For new window: propose_dimensions(0,0), set_position(0,0)
    → Focus the new window
    → manage_finish()

11. River sends: dimensions events for foot
    → UWM updates Window.width/height

12. River sends: render_start
    → UWM does nothing (position already set during manage)
    → render_finish()

13. foot is visible on screen.
```

## What v0.1 Will NOT Have

- No layout engine — windows at (0,0), auto-sized
- No focus movement keys — only Super+Space
- No close binding — only Super+Space
- No multiple output handling — just tracking
- No pointer bindings
- No interactive operations
- No IPC
- No configuration
- No keybinding beyond Super+Space

## Dependencies

- `wayland-client` (system library) — Wayland protocol client
- `xkbcommon` (system library) — for `xkb_keysym_t` constants (XKB_KEY_space, etc.)

Build system: Zig (via C-transpiled headers or zig-wayland bindings)
